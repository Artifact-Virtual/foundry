Foundry Courier

Darknet Continuum Toolkit
Carrier-agnostic, field-resilient, minimal.

  • Move signed blockchain transactions over radio, mesh, SMS, sneakernet, or any data carrier.
  • Text-based, portable, and runs on minimal hardware (USB, microSD, old phones, smartwatches with terminal access).
  • Encode/decode with CRC32 and parity for error correction.
  • Broadcast to Bitcoin and Ethereum networks, or via HTTP/SMS gateway.
  • Offline documentation and universal CLI/TUI interface.
  • Designed for extreme resilience and operational security.

Project and documentation: github.com/Artifact-Virtual/foundry

Getting Started
# 1. Install Python 3.11+ and dependencies python -m venv .venv && .venv\Scripts\activate pip install -r requirements.txt # 2. Encode a signed transaction python courier_cli.py encode-tx --hex <SIGNED_TX_HEX> --output frames.txt # 3. Decode frames python courier_cli.py decode-frames --input frames.txt --output recovered.hex # 4. Broadcast to Ethereum python courier_cli.py push-eth --hex <SIGNED_TX_HEX> --rpc-url <ETH_RPC_URL> # 5. Broadcast to Bitcoin python courier_cli.py push-btc --hex <SIGNED_TX_HEX> --rpc-url <BTC_RPC_URL> --user <USER> --password <PWD>
Code Example: Encode/Decode in Python
from courier.foundry_courier import encode_frames, decode_frames # Encode frames = encode_frames('deadbeef...') # Decode raw = decode_frames(frames)
Troubleshooting
  • Ensure Python 3.11+ is installed and your virtual environment is activated.
  • Check that all dependencies in requirements.txt are installed.
  • For network errors, verify your RPC URLs and credentials.
  • For SMS/HTTP gateway, ensure Flask and web3 are installed and the gateway is running.
  • Consult commands.lib for all CLI commands and usage.
Project Philosophy
  • Text-based first: Works on any OS, terminal, or device (including Termux, smartwatches, old phones).
  • Portable Python: Bundle on USB/microSD, run anywhere with Python or as a standalone executable.
  • Minimal dependencies: No heavy libraries, easy to audit and maintain.
  • Offline docs: All commands in commands.lib for field reference.
  • Universal UI: TUI/CLI for maximum compatibility; QR/barcode support possible for smart devices.
  • No private keys ever: Always sign offline, only move signed payloads.
  • Legal & safe: Respect radio laws, never transmit keys, repeat for reliability.
